# Specialty Workshops

Specialty workshops provide focused, deep-dive content beyond the core progression ([Foundations](../foundations/) → [General](../general/) → [OtterWorks](../otterworks/)).

They are organized into two categories:

## Technical Domains

Collections of modules targeting **specific tech stacks and engineering challenges**. A domain may contain multiple workshops, each covering a different technology or migration path within that domain.

| Domain | Workshops |
|--------|-----------|
| [Modernization](technical-domains/modernization/) | COBOL, Legacy, .NET Cloud-Native, Framework Upgrades, Data Source Migration, Platform Decomposition |
| [Security](technical-domains/security/) | Compliance, Enterprise Automation |
| [AI & Automation](technical-domains/ai-and-automation/) | Agentic AI |

## Job Families

One workshop per job family — **broad coverage of the role's engineering concerns**. These workshops span multiple technical domains to reflect the full scope of what that job family encounters day-to-day.

| Job Family | Workshops |
|------------|-----------|
| [Development](job-families/development/) | App Maintenance, Feature Development |
| [Digital Engineering](job-families/digital-engineering/) | CI/CD, cloud infrastructure, observability |
| [Quality Assurance & Engineering](job-families/quality-assurance-and-engineering/) | Engineering, Engineering + Security |
