# Evaluation: B3 — Dependency Lifecycle Campaign

## Blocking Criteria

- [ ] Participant scoped the campaign: listed services with languages and dependency files
- [ ] A Playbook was created for per-service dependency auditing
- [ ] At least 2 child sessions were launched (or simulated with sequential sessions)
- [ ] Child sessions produced PRs or findings reports

## Recommended Criteria

- [ ] At least 3 services were audited across different language ecosystems
- [ ] Findings were correctly categorized (CRITICAL/HIGH/LOW)
- [ ] CRITICAL findings (e.g., report-service Guava CVEs, collab-service lodash) were identified
- [ ] Participant can explain the divide-and-conquer pattern and its scaling properties
- [ ] Participant discussed scheduling the campaign as recurring automation
