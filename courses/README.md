# Courses

Reading-first **learning enablement** — work through these in sequence. Unlike the hands-on [`workshops/`](../workshops/) and [`labs/`](../labs/) (choose-your-own-adventure, pick what fits), courses are **low-optionality**: you complete them in order, and which track you take depends on your role.

No coding required. Each section is reading with light, optional exploration activities.

## Progression Model

```
Level 1: foundations/        ← Everyone starts here (concepts + product features)
    ↓
Level 2: tracks/    ← Role-specific (sales / solutions / engineering)
    ↓
Practice: ../workshops/      ← Apply it hands-on (general, otterworks, by-tech-domain, by-tech-role)
```

## Directory Structure

```
courses/
├── foundations/             ← Level 1: shared base — everyone completes this first
│   ├── concepts/            ← cloud vs. local, event-driven, orchestration, task selection
│   └── product/             ← feature tours by cloud / local
└── tracks/         ← Level 2: role-specific training
    ├── sales/               ← operating model, cost structures, engagement capture
    ├── solutions/           ← system design, SDLC integration, live walkthroughs
    └── engineering/         ← prompt engineering, orchestration, troubleshooting
```

## Level 1: Foundations

| Course | Focus | Duration |
|--------|-------|----------|
| [Foundations](foundations/) | Cloud agent concepts, product features, platform mental model | 2-3 hours |

Everyone starts here. Builds the mental model — cloud vs. local agents, event-driven reactions, orchestration, and task selection — before any hands-on work.

## Level 2: Tracks

Role-specific reading that builds on Foundations. Pick the track for your role.

| Track | Persona | Duration |
|-------|---------|----------|
| [Sales](tracks/sales/) | Sales Engineers, AEs, Practice Leads | 4-5 hours |
| [Solutions](tracks/solutions/) | Solutions Engineers, Architects | 8-10 hours |
| [Engineering](tracks/engineering/) | Engineers, Developers | 6-8 hours |

See the [Tracks README](tracks/) for the full per-track breakdown and content modalities.
